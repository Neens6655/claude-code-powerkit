## Svelte Architecture

### Reactive Component Framework
- **$: syntax**: Computed values and side effects
- **Automatic dependency tracking**: Reactive statements
- **Component architecture**: Single-file components with scoped CSS
- **Event handling**: on: directives and createEventDispatcher
- **Built-in animations**: transition directives, spring, tweened

### Component Composition
- **Named and scoped slots**: Flexible layout patterns
- **Teleport**: Cross-DOM boundary rendering
- **Conditional rendering**: #if, #each, #await blocks
- **Two-way binding**: bind: for forms and components

### Store Management
- **Writable stores**: Mutable global state
- **Readable stores**: External data sources
- **Derived stores**: Computed from other stores
- **Custom stores**: Business logic encapsulation

### SvelteKit Full-Stack
- **File-based routing**: +page.svelte, +layout.svelte conventions
- **Load functions**: Server and universal data loading
- **Form actions**: Progressive enhancement with server mutations
- **Rendering modes**: SSR, SSG, CSR, and hybrid strategies
- **API routes**: Server-side handlers with +server.ts

### Performance Optimization
- **Compile-time optimization**: No virtual DOM overhead
- **Minimal runtime**: Efficient DOM manipulation
- **Code splitting**: Manual and route-based strategies
- **Prerendering**: Static site generation for fast delivery
- **Virtual scrolling**: Large list rendering
